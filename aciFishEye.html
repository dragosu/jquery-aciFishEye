Here is the 'fixed' size demo, the <li> keep the same size and the image grow/shrink in the space of the <li> element.
Disable JavaScript in your browser to see it in action with CSS. I have made the CSS to use transitions if available (see the 'aciFishEye-nojs.css' CSS file).
The 'aciFishEye-nojs.css' file must be included after the CSS for the FishEye and only when JavaScript is not available (see the <noscript> tag).
The right side 'box' show you (by the red borders) how elements grow and shrink depending on the mouse position (and how 'margin-left'/'margin-top' CSS rules are used).
The example here implements the 'display' callback to center the tooltips on buttons (for the horizontal FishEye). Note that you probably can make this with some extra HTML/CSS but I took the easy path and also wanted to show you how the 'display' callback can work.

Note that because of the plugin flexibility you can have your own HTML structure (see plugin init options) and your own CSS rules. I recommend you to first implement the CSS version (with JavaScript disabled) and then go with the JavaScript version.
With the 'display' callback you can also show/hide things outside of the FishEye container and not only how it's implemented by default.

The minimum button size is read from the CSS rules, the maximum you set using plugin init options. Also, the horizontal FishEye must have the 'float' CSS property set to 'left' or 'right' on the 'block' element (<li> in our case, see plugin init options).

You can see a demo here for the dynamic version.
